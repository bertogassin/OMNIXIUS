# OMNIXIUS — Сверхстабильный стек на десятилетия

Единый набор языков и технологий для долгосрочного масштабного проекта с большим количеством пользователей. Ориентация: **надёжность, безопасность, масштабируемость** на десятилетия.

**Порядок языков — лучшие в начале**, чтобы не уходить в болото и не переделывать потом.

---

## 1. Порядок языков (первый = приоритет)

**Rust → Go → Spark → Swift → Kotlin → TypeScript → Python → C++/C → SQL → Bash/PowerShell**

(Solidity убран 100% — Rust важнее, блокчейн только на Rust.)

При выборе языка для новой задачи брать по этому порядку: первый подходящий по направлению — использовать.

---

## 2. Ядро (лучшие в начале)

| Язык   | Назначение |
|--------|------------|
| **Rust** | Высокопроизводительные и безопасные сервисы, блокчейн (ядро), микросервисы, игры, критический backend. Безопасность памяти, скорость, надёжность. |
| **Go**   | Backend, высоконагруженные API, микросервисы, масштабируемость, сеть. Простота, один бинарник, предсказуемый деплой. |
| **Spark**| Big Data, аналитика, обработка огромных объёмов данных. |
| **Swift**| iOS и macOS приложения, мобильные игры, клиентская часть Apple-экосистемы. |

---

## 3. Дополняющие языки (полное покрытие платформ)

| Сфера | Язык | Зачем |
|-------|------|--------|
| **Android / мобильные клиенты** | **Kotlin** | Полная мобильная платформа: Swift (iOS) + Kotlin (Android). Долгосрочная поддержка Google. |
| **Web frontend** | **TypeScript** | Весь веб только на TypeScript (React/Vue). JS и HTML — убрать по максимуму; оставить минимально только там, где технически неизбежно (например один bootstrap для SPA). |
| **Скрипты / ML / AI / ETL / DevOps** | **Python** | Обработка данных, AI/ML, генерация контента, автоматизация, скрипты для DevOps. |
| **Критичные микросервисы / игры** | **C++ / C** | Игровые движки, высокопроизводительные ядра, реальное время (FPS, MMO). Скорость и контроль. |
| **Инфраструктура / DevOps** | **Bash / PowerShell** | Автоматизация серверов, развёртывание, CI/CD, кроссплатформенные скрипты. |
| **Данные / аналитика** | **SQL / PL-SQL** | Базы данных, транзакции, аналитика, хранение пользователей и логов. |

---

## 4. Блокчейн: только Rust

- **Rust** — весь блокчейн: ноды, протокол, ZKP, смарт-контракты (WASM/интеграции), высоконагруженные компоненты. Solidity не используем: Rust важнее, будущих проблем меньше.

---

## 5. Сводка по направлениям

| Направление | Языки |
|-------------|--------|
| Критические ядра / Backend | **Rust** + **Go** |
| Мобильные клиенты | **Swift** (iOS) + **Kotlin** (Android) |
| Web frontend | **TypeScript** (React/Vue); JS/HTML убрать по максимуму |
| Big Data / аналитика | **Spark** + **Python** + **SQL** |
| Блокчейн | **Rust** (100%, без Solidity) |
| Высокопроизводительные игры / движки | **C++** |
| Скрипты / автоматизация | **Python** + **Bash** / **PowerShell** |
| Данные и персистентность | **SQL** / PL-SQL |

---

## 6. Долгосрочность

- **Rust и Go** — ядро, микросервисы, блокчейн → десятилетия без смены парадигмы.
- **Swift + Kotlin** — мобильные платформы → долгосрочная поддержка Apple и Google.
- **TypeScript** — frontend → строгая типизация; JS/HTML только минимально где неизбежно.
- **Python + Spark + SQL** — аналитика и данные → гибкость для ML, AI, ETL.
- **Rust** — блокчейн целиком (Solidity не используем).
- **C++** — высокопроизводительные игры и движки → скорость и контроль.
- **Bash / PowerShell** — инфраструктура → стандарт автоматизации и CI/CD.

Стек покрывает: фронтенд, мобильный, backend, критические микросервисы, блокчейн, аналитику, игры, DevOps. Каждый язык выбран на максимально долгий срок.

---

## 7. Связь с репозиторием

- **backend-go/** — Go API, микросервисы.
- **web/** — основной веб на TypeScript + React (app/ и js/ убрать/заменить по мере миграции).
- Новую серверную логику не вводить на Node; использовать **Go** или **Rust**. Новый фронт только на **TypeScript**.
